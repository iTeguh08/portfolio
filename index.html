Simple input output report card

This project is a simple program created to manage and print student report card scores. Completed during my 10th grade, this project served as the final project after dozens of assignments given by the teacher. The goal of this project was to provide practical experience in C++ programming by utilizing various fundamental concepts, such as data types, arrays, and functions.

CRUD schedule list ferry boat

Visit Site

In grade 10, I received a challenge from my Software Engineering (RPL) teacher to create a ferry scheduling system with features for creating, reading, updating, and deleting (CRUD).

The Ferry Scheduling System is a comprehensive web application, built with a robust tech stack combining Laravel as the backend framework, Vue.js for the frontend interface, and MySQL for database management.

This system demonstrates a complete CRUD (Create, Read, Update, Delete) implementation, showcasing modern web development practices and efficient data handling. The interface features a clean, user-friendly design with a prominent "BUAT TIKET" (Create Ticket) button positioned at the top right corner for easy access. The core functionality revolves around a well-structured table displaying crucial information including passenger names, route details, and timestamp data. Each entry in the system is carefully tracked with both creation and modification timestamps, ensuring full transparency in data management.

The system's CRUD operations are intuitively implemented through a three-button action system - "Tampilkan" (Show) for reading data, "Ubah" (Edit) for updating entries, and "Hapus" (Delete) for removing records - each color-coded in blue, green, and red respectively. The Vue.js frontend provides smooth, responsive interactions, while Laravel handles secure API endpoints and business logic on the backend. Data management is streamlined through Laravel's elegant ORM (Object-Relational Mapping), enabling efficient interaction with the MySQL database. Through the "BUAT TIKET" function, new schedules can be added to the system, while existing entries can be modified or removed as needed.

Each modification is automatically timestamped, providing a clear audit trail of changes made to the schedule. The responsive design implemented with Vue.js ensures the application adapts seamlessly across different devices, while Laravel's robust security features protect against common web vulnerabilities. This project exemplifies a well-architected full-stack application, demonstrating proficiency in modern web development frameworks and database management.

Heroes Mobile Legends Website

Visit Site

This project is a Mobile Legends gallery website that provides comprehensive information about the heroes in the game. On this website, users can view details about heroes such as their name, skins, roles, and abilities, which are displayed as icons. When an ability icon is clicked, a pentagon background activates, and a detailed description of the ability is displayed.

This project incorporates several features, including:

Scroll Content with TranslateX - Hero elements are displayed as users scroll down, creating a smooth transition effect where the heroes appear gradually. Popup - When the arrow button is pressed, a popup panel appears showing the hero's match results and abilities. Active and Remove Active - This feature is used to manage the highlight of the selected ability, ensuring that only one ability can be active at a time. Sound - A dedicated button allows users to play the hero's voice lines, controlled by play/pause buttons so users can listen to the voice lines of their chosen heroes.

This project was created during grade 11, combining HTML, CSS, and JavaScript to deliver an interactive experience. By adding features like audio, element transitions during scrolling, and popup displays, this website offers a rich visual and interactive experience for users who want to learn more about Mobile Legends heroes.
